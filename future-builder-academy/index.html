Course promise

Not passive AI users. Real builders.

The course starts with the attached Chew Gang foundation, then expands into computer fluency, coding, electronics, AI literacy, controlled automation, business/value creation, and a final useful project.

Think clearly

Students learn growth mindset, first principles, systems thinking, and how to break problems into true pieces.

Build physically

Electronics makes learning visible: sensors, LEDs, motors, servos, and low-voltage projects show cause and effect.

Use AI wisely

AI is taught as a learning partner and controlled tool, not an answer machine or uncontrolled agent.

Age bands

Same thesis, different depth.

Do not teach this the same way to every age. The course uses the same ladder but changes tools, autonomy, and risk.

Ages 7–10

Spark Builders

Curiosity, growth mindset, Scratch/block coding, safe circuits, drawing ideas, simple presentations, adult-guided AI.

  • No API keys
  • No autonomous agents
  • No public posting
Ages 11–13

Junior Builders

Microcontrollers, simple coding, project notebooks, AI literacy, source checking, websites, first useful prototypes.

  • Adult-controlled accounts
  • Sandboxed projects
  • More reflection and testing
Ages 14–18

Future Operators

Python/JavaScript, Arduino/ESP32, data, dashboards, AI assistants, controlled agents, entrepreneurship, capstone portfolios.

  • Structured outputs
  • Tool permissions
  • Logs and approval gates
Safety

Rules that keep the course serious.

Safety is not a separate lecture. It is built into electronics, AI use, internet use, project sharing, and automation.

Physical safety

  • Low voltage only unless supervised by qualified adults
  • No mains electricity
  • No blades, heaters, drones, or dangerous motors in beginner levels
  • Use fuses, limits, and manual shutoff on physical projects

AI and internet safety

  • No private personal info in AI tools
  • No AI messaging strangers
  • No spending or public posting without adult approval
  • Students must explain final work in their own words

Final capstone: build something useful for a real person.

Students choose a physical device, digital tool, AI-assisted helper, or small creative/business project. They must present the problem, user, build, failure, improvement, and next version.

Problem → User → Prototype → Test → Failure → Improvement → Presentation → Next version
