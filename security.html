Security & policy

Operate like a treasury, not a trader.

This program is designed to be operationally boring: clear authority, clear constraints, and clear reporting. The goal is resilient reserves — not experimentation.

Non‑negotiables

These rules exist to prevent the most common treasury failures: hidden leverage, custody mistakes, and policy drift.

Principle

The company is buying time and optionality. Any shortcut that increases fragility defeats the purpose.

Policy constraints
  • No rehypothecation: no lending, no yield products, no derivatives.
  • No discretionary sells: execution stays inside the policy; optional trims are predefined and capped.
  • Buffer first: cash runway is protected; contributions pause if buffer falls below target.
  • Separation of duties: the person who approves is not the person who executes.
  • Audit trail: every trade is recorded; every month is reported.
Recommended custody posture

Institutional custody or multisig with separation of duties. Avoid long‑term exchange custody.

Monthly report fields

A treasury program must be legible. Every month, the same dashboard — so leadership can govern without guessing.

Category Fields
Holdings BTC balance, USD cash balance, BTC‑eq, cost basis (policy-defined), custody location
Runway Cash buffer target, current buffer, months of runway (simple view), contribution pause/resume flags
Activity Trades executed, fees paid, execution mode, exceptions (if any) and approvals
Accounting Realized gains/taxes where applicable, reconciliations, documentation links
Governance Policy version, authorized signers, separation-of-duty checks, board notes/decisions

If you can’t explain your treasury position in 60 seconds, it’s too complicated.

Implementation checklist
  • Define buffer target and pause rules
  • Select custody approach (institutional or multisig)
  • Define authorized roles and approvals
  • Select execution mode
  • Set monthly reporting cadence
Common failure modes
  • Buying without a buffer policy
  • Leaving treasury assets on exchanges
  • Using leverage to “accelerate”
  • Changing rules mid‑cycle
  • Skipping reporting when volatile
