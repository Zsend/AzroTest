# Justice Grows go-live checklist

A beautiful site is not the launch gate. The launch gate is whether the organization can protect people, prove records, create real jobs, correct mistakes, and sustain operations.

## Safe to publish now: public test mode

- [ ] Existing AzroTest site backed up and rollback tested
- [ ] `config.js` remains in `local` mode
- [ ] Browser-test warning is visible on every interactive page
- [ ] No real personal or custody data used
- [ ] Homepage, Passport, talent profile, operations console, legal pages, 404, and mobile layouts reviewed
- [ ] GitHub Pages workflow passes
- [ ] Repository secret scan passes

## Required before real intake

- [ ] Operating entity, board oversight, and paid lived-experience advisory function established
- [ ] Brand, domain, trademark, fundraising, and charitable-solicitation review complete
- [ ] Privacy, employment, cannabis-regulatory, publication, public-records, and security counsel engaged
- [ ] Final privacy notice, terms, consent language, correction policy, methodology, data-retention schedule, and incident plan approved
- [ ] Cyber, D&O, general, employment, and professional/E&O insurance reviewed
- [ ] Production API deployed behind TLS; browser-local mode disabled
- [ ] Managed database, encrypted backups, key management, MFA, named operator accounts, least privilege, logging, alerting, and recovery tests complete
- [ ] CAPTCHA/bot defense, distributed rate limiting, dependency scanning, SAST, DAST, and penetration test complete
- [ ] WCAG 2.2 AA audit and assistive-technology testing complete
- [ ] High-risk correction freeze and 24-hour escalation staffed
- [ ] Candidate navigator and registry reviewer training complete

## Required before a public registry record

- [ ] Identity resolved without relying on name alone
- [ ] Current commitment and all material counts reviewed
- [ ] Cannabis attribution supported by operative records
- [ ] No violent offense identified statement uses the dated methodology; no personality label
- [ ] Current custody/release status checked against an authoritative source
- [ ] At least two documented sources, including an official court/commitment/records source
- [ ] Two distinct trained reviewers complete every required check
- [ ] Conflicts, detainers, enhancements, secondary counts, resentencing, and stale locator data resolved or disclosed
- [ ] Public-interest basis and/or consent documented
- [ ] Public fields minimized; exact location, contact data, medical information, family information, and release destination excluded
- [ ] Correction channel and source history available

## Required before an employer or job is published

- [ ] Employer identity, operating status, named hiring owner, and policy reviewed
- [ ] Fair Chance Employer Compact signed
- [ ] Position is open, funded, paid, and tied to a real manager
- [ ] Compensation, schedule, location, duties, benefits, growth path, and expiration documented
- [ ] State/local licensing and role-specific restrictions reviewed by qualified counsel or approved workflow
- [ ] No blanket exclusion, conviction browse, candidate fee, unpaid trial, coercive marketing, or second-chance wage discount
- [ ] Candidate-approved introduction and complaint escalation in place
- [ ] Employer agrees to outcome reporting and audit

## Required before claiming impact

- [ ] Metric definition, source event, owner, timestamp, and deduplication rule documented
- [ ] Starts require employer and candidate confirmation
- [ ] Retention is measured at defined 30/90/180/365-day points
- [ ] Wage, benefits, safety, advancement, and separation quality tracked—not starts alone
- [ ] Targets remain visually and analytically separate from actuals
- [ ] Small cohorts suppressed under the configured privacy threshold
- [ ] Revisions are logged; no retroactive metric inflation
