# AZRO Systems — Brand Copy Standard

## Core thesis
Build more Bitcoin through the cycle.

## Positioning
AZRO helps people build long-term Bitcoin without living in daily noise. The AZRO BTC Operating System handles steady weekly BTC building with one clear action, one suggested amount, optional trim guidance, and proof on chart versus a simple auto-buy baseline. The AZRO XRP Top/Bottom Indicator helps with major XRP bottom, top, and risk windows on the 1W chart so gains can be rotated back into long-term BTC.

## Product architecture
- The AZRO BTC Operating System = the core Bitcoin-building tool
- The AZRO XRP Top/Bottom Indicator = the XRP cycle tool for major 1W moves
- Use one or both, but keep the end goal the same: more long-term Bitcoin

## Allocation philosophy
- Public brand standard: BTC stays the majority of crypto capital.
- Default public range: roughly 70–90% BTC and 10–30% XRP.
- 50 / 50 is not the default. Treat it only as an outer-risk boundary for unusually aggressive operators who already maintain a separate untouched BTC core.
- Realized XRP gains should have a predefined path back into BTC or protected reserve capital.

## Voice
- Professional, plainspoken, proof-first
- Calm, serious, non-hype, anti-chaos
- Focus on long-term Bitcoin building, discipline, and clear weekly decisions
- Never frame AZRO as a day-trading room, leverage funnel, or get-rich-quick trading service

## Messaging rules
- Use “the AZRO BTC Operating System” or “the BTC Operating System” in running copy
- Use “the AZRO XRP Top/Bottom Indicator” or “the XRP Top/Bottom Indicator” in running copy
- Prefer language that normal people understand on first read
- Keep the end goal explicit: more long-term Bitcoin, less noise, and fewer emotional mistakes
