# AZRO brand copy standard — current site baseline

Core promise:
Build more Bitcoin. Cut the noise.

Core hierarchy:
- Bitcoin is the finish line.
- XRP can make the bigger move.
- The point of the move is still more Bitcoin.

Default buying logic:
- Most people should start with the AZRO BTC Operating System.
- Add the AZRO XRP Top/Bottom Indicator when you want structure around the bigger XRP cycle swings.
- Use XRP on its own only if a long-term Bitcoin plan already exists.

Copy rules:
- Plain English.
- Short sentences.
- Proof before hype.
- Explain what the user gets, how it works, and why it matters.
- Separate claims from proof.
- Never make undocumented performance promises.
- Never let XRP sound like the whole strategy.

Primary proof-safe language:
- One clear weekly BTC action.
- One suggested amount.
- Optional trim guidance.
- On-chart accountability versus a matched weekly-buy control.
- Weekly XRP bottom, top, and risk structure.
- Published handbooks, setup sheets, and validation notes.
