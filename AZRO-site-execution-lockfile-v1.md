# AZRO Systems — Site Execution Lockfile

Built from the full chat plus the updated brand handoff. Use this as the site-specific source of truth beside the deeper handoff.

## Non-negotiables
- World-class, premium, serious, balanced, consistent, responsive.
- Plain-English, emotionally strong, commercially alive.
- Bitcoin-first. BTC is the foundation, destination, and reserve asset.
- XRP is the amplifier / cycle helper. Never weak. Never an afterthought.
- Never use “signals.” Always use alerts / labels / confirmations / workflow.
- Do not invent new strategy jargon.
- Do not break layout, spacing, rhythm, or responsiveness.

## Core copy rules
- Short, direct, concrete. No hype. No fluff. No jargon.
- Use “you” and “your” naturally, but do not overuse them.
- Every page should read like a clear ad for the same system, not separate mini-brands.
- BTC language should communicate long-term accumulation, reserves, and protection.
- XRP language should communicate major cycle moves, readable alerts, and disciplined amplification.
- Preserve the founder’s original meaning and practical value props. Do not rewrite away the substance.

## Words and framing
- Say alerts, labels, confirmations, workflow.
- Do not say signals.
- BTC = foundation / destination / reserve.
- XRP = amplifier / bigger cycle / helper for the end-state.
- Strong proof language is good, but only if supportable.

## Homepage rules
- Hero must quickly explain the transformation:
  - capture major cycle moves
  - accumulate more Bitcoin
  - clearer weekly BTC guidance
  - optional trim logic
  - on-chart proof
  - readable XRP cycle alerts
- Home hero trust note under image card should be one line when possible.
- Home accordion titles should read like one strong ad.
- “Build stronger BTC reserves, not more chart time.” stays in the flow.
- Start-with-one section order: BTC, XRP, Both.

## Philosophy rules
- Hero must feel powerful, not abstract.
- BTC foundation / XRP amplifier hierarchy must be explicit.
- Page should explain why this matters now, why Bitcoin matters, and why calmer discipline beats noise.
- Accordion titles must read like one coherent premium ad.

## Products vs Features rules
- Products and Features must share the same section rhythm.
- Same divider treatment.
- Same centered text block under divider.
- Same image-card scale and balance.
- Same mini-card grid rhythm.
- The only meaningful difference:
  - Features keeps walkthrough / deeper explainer content.
  - Products stays more sales/product oriented.
- Keep one short meta line above each image card.
- Keep one short centered chart note below each image card.
- Avoid stacked section titles where shorter wording solves it.

## Pricing rules
- Hero must clearly mention lifetime and subscription.
- Subscription is lower-friction, not inferior.
- BTC remains the clearest starting point.
- “Not sure where to start?”, “How access works”, and “Pay with BTC or XRP.” should use the same accordion pattern.
- Direct crypto checkout can use wallet addresses + TXID confirmation flow.

## Resources rules
- Nav label can be “Proof”.
- Page should feel complete, trustworthy, and useful.
- Intro text between hero and document cards should be short and centered.
- Stay Updated and Support cards should be normal cards, not accordions.
- Utility rows should be centered and balanced.
- Core disclosures should look like the rest of the tile system.

## Footer rules
- Two-panel top row only: brand panel + signup panel.
- No redundant footer navigation blocks.
- Keep a simple inline lower row for legal/social/back-to-top.
- Signup copy must feel valuable and high-signal.
- Brand panel and signup panel must feel balanced and equal.

## Trial modal rules
- Keep the original visual direction.
- No internal scroll on desktop.
- Left-aligned intro copy is acceptable if it improves balance.
- Clear no-card / free-plan / next-step explanation.

## Layout rules
- Equal spacing around everything.
- Nothing should feel patched, off-center, or like a downgraded variant.
- Avoid stacked trust labels and stacked one-line meta where avoidable.
- Buttons, bullets, text blocks, and image cards must line up cleanly.
- Image cards on Products/Features should feel balanced left/right relative to the text block.
- Mobile must remain polished, centered, and high quality.

## Specific do-not-repeat mistakes
- Do not change approved copy direction without a reason.
- Do not keep reintroducing weak or abstract phrases.
- Do not remove key founder meaning from hero paragraphs.
- Do not make XRP sound small or unimportant.
- Do not let Products and Features drift apart visually.
- Do not stack headings awkwardly if shorter wording or better sizing solves it.


## Additional final design rules
- Hero trust line under image cards stays on one line whenever possible.
- Do not stack one-line product/feature section headings on desktop.
- Products and Features open sections should share the same two-column rhythm and vertical balance.
- In Products/Features open sections, the text block should feel vertically centered beside the image card.
- Keep intro bridge text between hero cards and content cards short, centered, and close to the next card block.
- Footer signup and brand panels should feel equal in visual weight.
- Short paragraphs should not be forced into overly narrow measure boxes on desktop.
