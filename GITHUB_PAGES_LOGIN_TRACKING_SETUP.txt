Reserve Standard — GitHub Pages Login Tracking Setup

GitHub Pages is static hosting, so it cannot store login events by itself. The easiest durable setup is:

1. Create a Google Sheet named: Reserve Standard Login Events
2. In the Sheet, open Extensions → Apps Script.
3. Paste the included file `github-pages-login-tracking-apps-script.gs` into Code.gs.
4. Click Deploy → New deployment → Web app.
5. Set:
   - Execute as: Me
   - Who has access: Anyone
6. Copy the Web app URL.
7. Open `site-config.js` and paste the URL here:

   loginTrackerEndpoint: "PASTE_GOOGLE_APPS_SCRIPT_WEB_APP_URL_HERE",

8. Publish the site to GitHub Pages.
9. From your own browser, visit:

   https://YOUR_DOMAIN_OR_GITHUB_PAGES_URL/?rs_owner=RS-OWNER-2026-8K4M

   This marks your current browser as the owner device. Future unlocks and return visits from that browser log `isOwner=true` and `ownerLabel=Zach`.

10. To clear the owner marker from your browser, visit:

   https://YOUR_DOMAIN_OR_GITHUB_PAGES_URL/?rs_owner=clear

What gets logged:
- gate_unlock events when someone enters the access code successfully
- authorized_visit events when a previously unlocked browser returns
- anonymous browser ID
- owner flag / owner label
- page path and title
- referrer
- viewport and screen size
- timezone
- user agent
- site version

What does NOT get logged:
- the access code
- names
- emails
- wallets
- payment information

Important: This is analytics labeling, not real authentication. A static GitHub Pages site cannot securely identify people by itself. For true customer identity, use a real auth provider or backend later.
