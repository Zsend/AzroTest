XRP Top/Bottom Detector

Key features designed to reduce noise and support rules-based, cycle-level decisions on XRP — evaluated on the weekly close.

Weekly-close confirmations

MAJOR and RADAR confirmations are designed to trigger on the weekly bar close (1W) — helping you make cycle-level decisions without staring at the chart.

EARLY + LIGHT context (optional)

EARLY labels can print intra-week as an early context layer; LIGHT labels add timing visibility between major turns. Leave them on for awareness, or turn them off for a cleaner, confirmation-only chart.

Risk context overlays

RISK and HIGH-RISK triangles add a conservative context layer. Optional controls can add a risk-colored XRP price line and (if enabled) market-context risk markers.

Designed for disciplined execution

Built for process, not prediction: confirmations at the weekly close, optional soft-confirm filtering, and quality-of-life controls that reduce noise and stress.

TradingView-native workflow

Install on TradingView, set alerts once, and follow your written plan. The script is distributed as an invite-only TradingView indicator after purchase.

Docs-first onboarding

Clear docs for setup, alerts, and disclosures. Start with the recommended baseline, then customize only after you’ve validated the workflow.

BTC Engine

A dashboard-first weekly operating system for Bitcoin: adaptive DCA, optional trims, and built-in proof vs auto-buy.

Dashboard-first workflow

Designed for a simple weekly routine: after the weekly close, read ACTION (BUY / HOLD / TRIM optional) and the suggested amount, then execute and log.

Execution modes

Choose a mode that matches your policy: Stack (buys only), Adaptive (optional trims + smoother re-entry), Event (reacts on major events), or Performance (aggressive preset).

Risk profiles + cash buffer

Set a target cash buffer and pick Conservative, Balanced, or Aggressive (BTC‑max) to shape how the engine deploys cash across cycle conditions.

Proof vs auto-buy

The dashboard tracks BTC‑equivalent vs a simple auto-buy baseline (with the same deposits) so you can compare discipline vs automation. Comparison metrics are not guarantees.

Major cycle event alerts

Optional alerts highlight rare high-signal windows (often near cycle inflections) so you can review the dashboard at the right time — without staring at charts.

Data hygiene + safety guard

Includes a robust weekly wick filter (multi-exchange median) and a safety guard that helps prevent contradictory “top + bottom” states on the same bar.
